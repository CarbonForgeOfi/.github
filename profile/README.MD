## üéØ What is CarbonForge?

CarbonForge is a comprehensive VPS hosting service managed entirely through Discord. It combines the power of Proxmox virtualization with an easy-to-use Discord bot interface, making VPS deployment and management accessible to everyone.

**Core Capabilities:**
- ‚úÖ **Instant Deployment** - Deploy VPS containers in minutes via Discord commands
- ‚úÖ **Full Lifecycle Management** - Create, manage, monitor, and delete containers
- ‚úÖ **Automated Billing** - Integrated invoicing and payment system
- ‚úÖ **Resource Monitoring** - Real-time CPU, RAM, and disk usage tracking
- ‚úÖ **Developer Ready** - Pre-configured with Python, Node.js, Docker, and Git
- ‚úÖ **File Management** - Upload and share files up to 20GB
- ‚úÖ **Network Tools** - Built-in diagnostics and speed testing

**Platform:**
- **Type:** Discord Bot (Closed Source)
- **Infrastructure:** Proxmox VE
- **Availability:** Exclusive to CarbonForge Discord Server
- **Status:** Production Ready

## üöÄ Getting Started

### For Users

1. **Join the Discord Server**
   - Join the [CarbonForge Discord Server](https://discord.gg/MSUAmVu7d9)
   - The bot is exclusive to this server

2. **Read the User Guide**
   - Check out the [User Guide](https://github.com/CarbonForgeOfi/USER-GUIDE) for complete instructions
   - Learn how to deploy your first VPS

3. **Deploy Your VPS**
   - Use `/price` to see available plans
   - Use `/deploy` to create your container
   - Follow the payment process

### For Staff

1. **Get Permissions**
   - Contact an Owner to get staff permissions
   - Use `/setperm` to grant roles

2. **Read the Staff Guide**
   - Review the [Staff Guide](https://github.com/CarbonForgeOfi/STAFF-GUIDE) for all commands
   - Understand your role and responsibilities

3. **Start Managing**
   - Use `/help` in Discord to see available commands
   - Follow best practices from the guide

### For Developers

1. **Access Developer Resources**
   - Request access to the [Developer Guide](https://github.com/CarbonForgeOfi/DEV-GUIDE) repository
   - Review code structure and architecture

2. **Set Up Development Environment**
   - Follow the deployment guide
   - Configure your local environment

3. **Start Contributing**
   - Read the code examples
   - Follow security best practices
